# Day5：タイトル画面とカウントダウンの日（7人×3時間用シナリオ／生徒用）

## 0. 今日のゴール

3時間で、次の3つができるようになることを目指します。

1. ゲームを起動すると **タイトル画面** が出る  
2. Enter キーでタイトルを閉じると、**3,2,1,Go! のカウントダウン** が流れる  
3. カウントダウンのあとにランゲーム本編が始まり、GameOver から **R キーで再スタート** できる  

今日は「**ゲーム全体の流れ（状態）**」を作る日です。  
「タイトル → カウントダウン → プレイ → GameOver → タイトルへ…」という  
一周のサイクルを `mode` という変数で管理します。

## 3. チェックポイント

### 3-1. タイトル画面

- [ ] ゲームを起動すると、真っ暗ではなくタイトルが出ている  
- [ ] タイトルにゲーム名と「[Enter]でスタート」のような案内が書かれている  
- [ ] タイトル中はスコアの代わりに、Day5 のゴールが HUD に出ている  

### 3-2. カウントダウン

- [ ] Enter キーでタイトルが消え、画面中央に「3」が出る  
- [ ] 1 秒ごとに「3 → 2 → 1 → Go!」と変わる（秒数はクラスで相談してOK）  
- [ ] 「Go!」のあと、自動でランゲームが始まる  

### 3-3. プレイ〜GameOver〜リスタート

- [ ] スペースキーでジャンプできる  
- [ ] 障害物に当たると GAME OVER になる  
- [ ] GameOver 中はジャンプしても意味がない  
- [ ] R キーで **もう一度** カウントダウンに戻れる  
- [ ] Enter キーで **タイトル画面** に戻れる

## 4. うまくいかないときのヒント

- 何も表示されないときは、`enter_title()` が `__init__()` から呼ばれているか確認する  
- スペースでジャンプできないときは、`_on_key_down()` の `key == 32` の条件を確認する  
- カウントダウンが一瞬で終わってしまうときは、`self.countdown -= dt` の扱いを先生と相談する  
- GameOver になってもスコアが増え続けるときは、`game_over()` と `update()` の関係を調べてみる  

分からないことがあったら、まずは **自分の役割の範囲** をもう一度読み直してみましょう。  
それでもつまずいたら、先生やまわりのメンバーに「どこが分からないか」を言葉にして相談してください。
